Generic Request (Auto routes)


Generic Request (Auto route) module helps you to automate requests processing system built with Generic Request. Requests can be moved automatically between stages under certain conditions (by the trigger).

This addon along with Generic Request (Actions) can provide extra tools that help to automate your helpdesk (service desk) system.

Pre-configurations:

  • Configure stages of the request type
  • Set up routes (link all the stages) of the request type

How to use:

  1. Open the Request Type→Routes page and select one of the desired route to auto-move the request.


  2. Click Triggers to add a new trigger. The Auto Only check box will make this route automatic only (moving requests manually is forbidden on this route).


  3. Click Create to create a new trigger. You can create several triggers on one route.
  4. On the form, select the desired trigger. The following triggers are implemented:
    • Auto: On Create - will automatically trigger after request creation
    • Auto: On Write - will automatically trigger when writing fields (you can specify these fields)
    • Cron: Daily - will periodically trigger, once per day
    • Cron: Hourly - will periodically trigger, once per hour


  5. If necessary, set the Trigger Condition. This condition will be checked before triggering.
  6. If more than one trigger is created on one route, they will be ordered by the Sequence. The smaller the number, the greater the priority.
